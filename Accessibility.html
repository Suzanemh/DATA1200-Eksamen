Reflection of accessibility testing

What is Accessibility Testing?

Technology is changing the way we live, but it needs to change how we access it, especially in the digital age that provides innovation and new opportunities to everyone who consumes it. It requires a lot of effort in creating a technology that can be used by ‘everyone’. Accessibility Testing is defined as a type of Software Testing performed to ensure that the application being tested is usable by people with disabilities like hearing, color blindness, old age and other disadvantaged groups (Hamilton, 2021; TestingXperts, 2021).

Why Accessibility Testing?

There are several reasons to test accessibility. Some of the reasons are:

  • Cater to market for Disabled People
  • Abide by Accessibility Legislations
  • Avoid Potential Law Suits

More information about these reasons can be found at testingxperts blogs post.

Furthermore, there are great benefits to accessibility testing. Among other things, that (1)Increases the website usage and market share, (2) Enhances the brand image, and (3) Makes the website accessible to society (TestingXperts, 2021).

Navigation is the way users find and traverse the different pages available on your site. For this reason, it is imperative that navigation is accessible. The "skip to content-link" makes our website more accessible for blind users. The link allows users who navigate with keyboard or screen readers to easily skip to the main content without having to go through the header and navigation-menu every time (W3Schools, n.d.). By their nature, links are tab-able and all keyboard users and screen readers will read them–so, if your navigation is coded with links (see the code to left), a screen reader should find when tabbing through the navigation list (STVDIO, 2021). The code on the left is inspired by Rascia (2016) which highlight code syntax in a website rather than adding a screenshot. This allows screen readers to read the entire code.

Use of heading and paragraph tags

                    <h2> What is Accessibility Testing?</h2>

Web pages must be readable by electronic “screen readers” — which are devices commonly used by the visually impaired to surf the internet (Smith, 2018). Screen readers use heading tagsto navigate and prioritize the content. The strategic use of heading tags will not only make the site more easily manageable by screen readers but it will also make it more optimized for search engines (Smith, 2018). Therefor, it is important that there is a connection between the heading and the paragraph tag, so the website is more likely to be universally designed.

Use ALT text for images

Screen readers can also read the text in the ALT tags in embedded image files. These tags usually contain a description of any images, infographics, or other non-text information on the page. The more descriptive the image, the better for the user (Smith, 2018).

Use of descriptive links

It’s important for link text to make sense without the surrounding sentences or content. The link text alone should convey the function and purpose of the link. Link text should also be unique and easy to speak out loud (Yale University, 2021). In that case, Avoid link text like “Click Here,” “More,” and “Read More.” These kinds of links can be confusing when a screen reader reads them out of context. In our website, we provided this by giving a descriptive link. Example of this is when we refer a link by writing:
"More information about these reasons can be found at testingxperts blogs post."

Avoid parallax effects

A parallax effect is a design trend where background elements move at speeds other than foreground elements as they scroll. We first wanted to use this effect in the homepage, but after reading our advice for designing websites, we dropped the effect. Because this type of movement is harmful to people with vestibular disorders and should be avoided or used with extreme caution and restraint (WCAG, 2021).

Color contrast
